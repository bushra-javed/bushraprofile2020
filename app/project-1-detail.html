frontent developoer

I create responsive websites that allow the user to experience your website in the best and most appropriate way suited to the device they are using. By working using progressive enhancement, your website is delivered with a responsive layout that can best make use of the space available on the smallest to largest devices. Cross browser compatibility is ensured by using feature detection so older browsers still in use today provide a solid experience, whilst modern browsers can go the next step and enhance a user’s time spent on your site. Writing semantic markup that is documented and easy to read means it can be maintained and scaled in the future, and allows co-workers to quickly work with it. Rather than create websites on a page-by-page basis, I instead consider each element of your screen-based designs as separate components that can exist elsewhere on a website; outside of the context of your design compositions. With these components I produce a style guide that acts as a document for a reusable and maintainable code base. All involved in a project can rely on a style guide to understand a project’s language and produce a more consistent user experience. My front-end build process involves the use of tools such as Sass, Gulp, and NPM to speed up development. I can quickly and efficiently join your team using continuous-integration with methods and tools such as Git, Jenkins, Agile/Scrum, and test/behaviour driven development.
